Fully Dressed Use Case:

5.1 Search for concert/performance genre

Use Case Name:Search for concert/performance genre
Use Case Id 5.1
Priority 5
LevelUser Goal
Primary Business Actory Customer
Other Participating Actors Database Administrator
Description Customer select the concert/performance genre, the system retrieve and display available concerts/performances under that genre.
PreconditionsCustomer invoked the search for concert option
Stakeholders & Interests
Customer:
Wants to be able to search for concert/performance genre of their choice and see what is available for them
Company:
Wants to draw in more customers to increase revenue for the company
Database Administrator:
Wants to be able to track number of users using the system each day
Typical Scenarios
  1. Customer selects the type of genre, e.g. K-Pop
  2. The system retrieve and displays the concerts available under the choosen genre
Extensions/ Alternative Scenarios
  • Concert/Performance Genre not available: customer chooses another genre
Conclusioncustomer successfully finds available concert/performance of their genre choice
Post Conditionscustomer can now search for location and date of their choosen concert/performance
Business Rules
  1. Customer invoked the search for concert option
Implementation Constraints
