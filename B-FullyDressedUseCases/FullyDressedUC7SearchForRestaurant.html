7. Search for Restaurant Type

Use Case Name:Search for Restaurant Type
Use Case Id 7.1
Priority 7
LevelUser Goal
Primary Business Actory Customer
Other Participating Actors Database Administrator
DescriptionCustomer selects a restaurant type they want to search,e.g. italian. The system retrieve and display the available restauarnt under the selected restaurant type
Preconditions
Stakeholders & Interests
Customer:
Wants to be able to search for available restaurant type
Company:
Wants to draw in more customers to increase revenue for the company
Database Administrator:
Wants to be able to track number of users using the system each day
Typical Scenarios
  1. Customer invokes the search for restaurant option
  2. Customer then selects the restaurant type of their choice, e.g. Italian
  3. The system retrieve and displays the available restauarnt under the choosen restaurant type
Extensions/ Alternative Scenarios
  • Restaurant Type not available: customer chooses another restaurant type
Conclusioncustomer successfully finds available restauarnt of their choice
Post Conditionscustomer can now book a table at the choosen restaurant
Business Rules
  1. Customer invoked the search for restaurant option
Implementation Constraints
