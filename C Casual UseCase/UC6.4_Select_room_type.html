Casual Use Cases:

h2>UC6.4: Select room type Use Case Number: 6.4
Use Case Name: Select room type
Success: When the customer invokes the search hotel option, it will display room types that customer can book such as single rooom, double room or family room.
Alternative Scenario:
The customer wishes to select room type that is unavailable, in which case customer can choose another room type or diffrent hotel.
The customer does not want to book for hotel rooms, in which case customer can directly go pay for the ticket or exit the system.
