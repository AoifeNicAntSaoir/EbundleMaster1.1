Fully Dressed Use Case:

UC6: Search for Hotel

Use Case Name:Book Hotel
Use Case Number 6
Priority 6
Source
Primary Business Actory Customer
Other Participating Actors System
DescriptionCustomer can search for available hotel.
Preconditions Customer must be on EbundleMaster website.
Stakeholders & Interests
Customer:
Wants to be able to search for hotel on the Ebundle System
Company:
Wants to make profit for the company
System:
Wants to be make sure that customer can successfully search for hotel
Typical Scenario
  1. Customer is at EBundleMater website.
  2. Customer invokes Search for Hotel
  3. Customer inputs the 4 requirements requested by the system to search for hotel (location,check-in/out date, type of room type) to search for hotels
  4. system searches for hotels
  5. The system retrieves and displays list of hotels.
Alternate Scenario
  1. Invalid inputs were put into the requirements fields to search for hotels.
  2. Customer searched for a hotel that is not available.
  3. Customer does not want to search for hotel, than Customer can exit the system.
ConclusionsCustomer searched for hotel at EBundleMaster and the system retrieves and displays list of hotels.
Post ConditionsCustomer can now book a hotel that was retrieved and displayed.
