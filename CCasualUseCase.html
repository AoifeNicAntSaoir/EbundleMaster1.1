Casual Use Cases:

UC4: Add to cart

Use Case Number: 4
Use Case Name: Add to cart
Success: Customer most successfully logged into their account to add item to cart.Customer can add item to cart when they have identified an item they wish to purchase by invoking the add to cart option.
Alternative Scenario: When Tickets has been sold out, in which case customer will not be able to add the concert ticket to cart and will have to search for another concert or exit the system.
When a concert has been cancelled, in which case customer will not be able to add the concert and will have to search for another concert or exit the system.
When a hotel is unavailable, in which case customer will not be able to book hotel and will have to search for another hotel or exit the system.


UC5.4: Book Concert

Use Case Number: 5.4
Use Case Name: Book Concert
Success: Customer most successfully logged into their account and added concert tickets to cart to book a concert.
Alternative Scenario: When Tickets has been sold out, in which case customer will not be able to book the concert and will have to search for another concert or exit the system.
When a concert has been cancelled, in which case customer will not be able to book the concert and will have to search for another concert or exit the system.


UC6: Search Hotel

Use Case Number: 6
Use Case Name: Search Hotel
Success: Customer does not have to be logged on to their account to search for hotel. The customer can search for hotel when they invokes the search hotel option on EBundleMaster, customer will be shown four options for customer to input, their location, check-in date, check-out date and room type.
Alternative Scenario:
When there is no available hotel, the customer can search for other hotels nearby.
The customer does not want to book for hotel rooms, in which case customer can directly go pay for the ticket or exit the system.


UC6.1: Select hotel location

Use Case Number: 6.1
Use Case Name: Select hotel location
Success: When the customer invokes the search hotel option, it will display a location input box for customer to input location they wish to stay, the system will search for hotel.
Alternative Scenario:
The customer enters a invalid location, in which case customer must re-enter valid location. The customer does not want to book for hotel rooms, in which case customer can directly go pay for the ticket or exit the system.
