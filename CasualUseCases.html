Casual Use Cases:

UC 1: Account Sign up

Use Case Number: 1
Use Case Name: Account Sign-Up
Success: Customer successfully creates their account
Alternative Scenario:
  • Existing Account: A customer already has an existing account with the corresponding email address
  • The system cannot generate a unique user ID number - outputs an error message to the user
  • Invalid password: customer is asked to re-enter their password.
  • Invalid email: The system asks the customer to enter a valid email address in the format: username @ domain .ext
  • Validation link expires - the customer must go through a forgot password process again

UC 4: Search for concert/performance date

Use Case Number: 4
Use Case Name: Search for concert/performance date
Success: Customer is able to find date of a concert/performance
Alternative Scenario:
  • No concerts/performances available on date specified: The system returns with an error message & suggests other events
  • Filtered dates do not return a search: The systems suggests other events
  • The customer chooses not to look at concert event: The system does nothing else

UC 5: Search for concert/performance location

Use Case Number: 5
Use Case Name: Search for concert/performance location
Description: Customer selects a location for concert/performance.
Alternative Scenario: Location not available: customer must choose another location available

UC 7: Search for Transport

Use Case Number: 7
Use Case Name: Search for Transport
Description: A customer needs to choose:
  • Type of Transport
  • Date & time of departure
  • Date & time of Arrival
  • Location of venue
  • Location of hotel
Alternative Scenario:
  • No Transport for selected options chosen
  • No transport options are available
