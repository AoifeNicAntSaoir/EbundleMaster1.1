Fully Dressed Use Case:

UC4: Add to cart

Use Case Name:Add to cart
Use Case Number 4
Priority 3
Source
Primary Business Actory Customer
Other Participating Actors System
DescriptionCustomer can add items to cart.
Preconditions A customer must log on first to add items to cart.
Stakeholders & Interests
Customer:
Wants to be able to add items to cart
Company:
Wants to make profit for the company
System:
Wants to be make sure that customer can successfully add item to cart
Typical Scenario
  1. Customer adds item to cart by invoking add to cart/book now
  2. System adds the items to cart
  3. System Selects view cart
  4. System displays all items that are added to cart
Alternate Scenario
ConclusionsItem has been added to cart.
Post Conditions Customer can view cart and advance to payment procedure.
Business Rules Customer must have an account and logged on to add item to cart.
Implementation Constraints
